EventBus.gd
Назначение: Глобальная шина сигналов для связи между системами. Сигналы сгруппированы по вложенным классам (Game, Entity, Player, Relationship, Dialogue, Combat, Animations, UI, Actors, Flags, System). Реализует паттерн «Наблюдатель» и обеспечивает слабую связанность компонентов. Содержит подгруппы боя (enemy, decision, attack, dodge, block, ability) и слой совместимости со старыми сигналами.
Классы/Секции сигналов:
GameSignals
Управление глобальными состояниями и переходами игры.
•	state_changed(new_state: int, old_state: int) — изменение глобального состояния игры
•	paused(is_paused: bool) — логическая пауза игры
•	world_requested() — запрос перехода в режим WORLD
•	dialogue_requested(timeline_name: String) — запрос запуска диалога
•	menu_requested() — запрос открытия меню
•	cutscene_requested(cutscene_id: String) — запрос катсцены
•	game_over_requested() — запрос GAME_OVER
•	decision_point_activated() — активация точки решения (замедление времени)
•	decision_point_deactivated() — деактивация точки решения (нормальное время)
EntitySignals
Общие сигналы для всех живых сущностей (Player, Actor, Enemy).
•	health_changed(entity: Node, new_value: int, old_value: int, max_value: int) — изменение здоровья
•	damage_taken(entity: Node, amount: int, damage_type: int, source: Node, is_critical: bool) — получение урона
•	healed(entity: Node, amount: int, new_health: int) — лечение
•	died(entity: Node) — смерть сущности
•	mana_changed(entity: Node, new_value: int, old_value: int, max_value: int) — изменение маны
•	stamina_changed(entity: Node, new_value: int, old_value: int, max_value: int) — изменение выносливости
•	interaction_started(entity: Node, activator: Node) — начало взаимодействия
•	interaction_ended(entity: Node) — завершение взаимодействия
PlayerSignals
Специфичные сигналы игрока.
•	level_up(new_level: int, stat_increases: Dictionary) — повышение уровня
•	experience_gained(amount: int, new_total: int) — получение опыта
•	equipment_changed(slot: String, old_item: String, new_item: String) — смена экипировки
RelationshipSignals
Нарративная система «Доверие / Воля».
•	trust_changed(new_value: int, delta: int) — изменение доверия
•	will_changed(new_value: int, delta: int) — изменение Воли
•	will_used(amount: int, remaining: int) — использование Воли
•	trust_effect_applied(effect_type: String, multiplier: float) — применение эффекта от доверия
DialogueSignals
Интеграция с Dialogic 2.0.
•	started(timeline_name: String, npc: Node) — начало диалога
•	ended() — завершение диалога
•	choice_selected(choice_index: int, choice_text: String) — выбор игрока
•	requested(timeline_name: String, npc_id: String) — запрос диалога
•	change_trust(amount: int) — изменение доверия
•	set_flag(flag_name: String, value: Variant) — установка флага
•	use_will(amount: int) — использование Воли
•	start_battle(npc_id: String) — запуск боя из диалога
CombatSignals
Боевая система реального времени с подгруппами.
Управление боем:
•	started(enemies: Array) — начало боя
•	ended(victory: bool) — завершение боя
•	combat_state_changed(old_state: String, new_state: String) — смена состояния боевого FSM
•	player_stunned(is_stunned: bool) — изменение стана игрока
EnemySignals (Combat.enemy):
•	health_changed(enemy: Node, current_hp: int, max_hp: int) — изменение HP врага
•	died(enemy: Node, experience: int) — смерть врага
•	hit(enemy: Node, damage: int, is_critical: bool) — попадание по врагу
DecisionSignals (Combat.decision):
•	point_triggered(enemy: Node, trigger_data: Dictionary) — активация точки решения
•	made(enemy: Node, choice: String) — выбор игрока
•	ui_closed() — закрытие UI точки решения
•	dialogic_made(choice: String) — выбор через Dialogic
•	combat_to_dialogue_requested(enemy: Node) — запрос перехода из боя в диалог
•	point_requested(enemy: Node, trigger_data: Dictionary) — запрос точки решения
•	transition_to_dialogue(enemy: Node, timeline: String) — переход в диалог
•	enemy_spared(enemy: Node) — враг пощажён
AttackSignals (Combat.attack):
•	basic_requested() — запрос базовой атаки
•	basic_started() — начало атаки
•	basic_hit(combo_step: int, enemies_hit: int) — попадание атаки
•	combo_reset() — сброс комбо
•	combo_window_opened() — начало окна комбо
•	combo_window_closed() — конец окна комбо
DodgeSignals (Combat.dodge):
•	requested(direction: Vector2) — запрос уворота
•	started() — начало уворота
•	completed() — завершение уворота
BlockSignals (Combat.block):
•	started() — начало блока
•	ended() — конец блока
•	active() — блок активен
•	reduced_damage(damage_after_block: int) — урон после блока
•	broken() — блок сломан
AbilitySignals (Combat.ability):
•	slot_pressed(slot_index: int) — нажатие слота способности
•	aiming_started(slot_index: int) — начало прицеливания
•	aiming_cancelled() — отмена прицеливания
•	target_confirmed(target_position: Vector2) — подтверждение цели
•	cast_started(ability: Resource) — начало каста способности
•	cast_completed() — завершение каста
•	animation_started(animation_name: String, duration: float) — анимация способности
AnimationSignals
•	requested(target: Node, animation_name: String, duration: float) — запрос анимации
•	started(target: Node, animation_name: String) — начало анимации
•	finished(target: Node, animation_name: String) — завершение анимации
•	interrupted(target: Node, animation_name: String) — прерывание анимации
UISignals
•	menu_opened(menu_type: String) — открытие меню
•	menu_closed() — закрытие меню
•	notification(text: String, duration: float) — уведомление
•	hud_update_required() — запрос обновления HUD (также принимает Dictionary с данными, см. AbilityComponent)
ActorsSignals
•	mode_changed(actor: Node, new_mode: String, old_mode: String) — смена режима актора (world/battle)
•	interaction_started(actor: Node) — устаревшее событие начала взаимодействия (заменено Entity.interaction_started)
•	died(actor: Node) — устаревшая смерть актора (заменено Entity.died)
•	interaction_requested() — устаревший запрос на взаимодействие
FlagsSignals
•	flag_changed(flag_name: String, value: Variant) — изменение глобального флага
SystemSignals
•	settings_changed(setting_name: String, value: Variant) — изменение настроек
•	scene_changed(old_scene: String, new_scene: String) — смена сцены
Экземпляры классов:
•	Game, Entity, Player, Relationship, Dialogue, Combat, Animations, UI, Actors, Flags, System — синглтоны групп сигналов
•	Подгруппы Combat: enemy, decision, attack, dodge, block, ability
Методы:
•	_ready() — инициализация, очистка старых подключений, отладочный вывод
•	_clear_all_connections() — перебор всех групп/подгрупп и отключение всех сигналов
•	_print_debug_info() — краткая информация о структуре EventBus
•	_get_property_list() — добавляет служебное свойство для редактора
•	_get(property: StringName) — перехват обращений к устаревшим сигналам (Player_*), выводит предупреждение
Совместимость:
•	Добавлен слой миграции: при обращении к Player_damage_taken, Player_hp_changed, Player_mp_changed, Player_stamina_changed выводится предупреждение «используйте EventBus.Entity.*»
Зависимости: Нет (корневой синглтон, загружается через автозагрузку).

GameStateManager.gd
Назначение: Управляет глобальными состояниями игры (WORLD, DIALOGUE, BATTLE, MENU, CUTSCENE, GAME_OVER). Проверяет допустимость переходов, изменяет time_scale/paused и рассылает сигналы в EventBus.
Классы/Секции:
Enum GameState
•	WORLD — свободное перемещение по миру
•	DIALOGUE — активен диалог
•	BATTLE — бой в реальном времени
•	MENU — открыто меню
•	CUTSCENE — катсцена
•	GAME_OVER — конец игры
Переменные:
•	_current_state: GameState — текущее состояние
•	_previous_state: GameState — предыдущее состояние
•	eb: EventBus — ссылка на глобальную шину
Методы:
•	func _ready() -> void — настройка подключений, вывод текущего состояния
•	func _setup_event_bus_connections() -> void — подписка на сигналы EventBus.Game и EventBus.Combat
•	func change_state(new_state: GameState, force: bool = false) -> void — смена состояния с проверкой матрицы переходов
•	func _is_transition_allowed(from_state: GameState, to_state: GameState) -> bool — матрица допустимых переходов
•	func _handle_state_change_effects(new_state: GameState, old_state: GameState) -> void — изменение Engine.time_scale, get_tree().paused и посылка Game.paused
•	func _on_transition_to_world_requested() -> void — переход в WORLD
•	func _on_transition_to_dialogue_requested(timeline_name: String = "") -> void — переход в DIALOGUE и логирование имен диалогов
•	func _on_transition_to_battle_requested(enemies: Array = []) -> void — переход в BATTLE, лог количества врагов
•	func _on_transition_to_menu_requested() -> void — переход в MENU
•	func _on_transition_to_cutscene_requested(cutscene_id: String) -> void — переход в CUTSCENE
•	func _on_transition_to_game_over_requested() -> void — переход в GAME_OVER
•	func _on_dialogic_decision_made(choice: String) -> void — обработка выбора из Dialogic («to_combat»/«to_world»)
•	func _on_entiry_died(entity: Node) -> void — реагирует на смерть игрока (группа player), возвращает в WORLD
Публичные методы:
•	func get_current_state() -> GameState
•	func get_current_state_name() -> String
•	func can_player_move() -> bool — можно ли двигаться (WORLD/BATTLE)
•	func is_ui_blocking() -> bool — активно ли блокирующее UI-состояние
•	func is_in_combat() -> bool
•	func is_dialogue_active() -> bool
Подписка на EventBus:
•	eb.Game.world_requested
•	eb.Game.dialogue_requested
•	eb.Combat.started
•	eb.Game.menu_requested
•	eb.Game.cutscene_requested
•	eb.Game.game_over_requested
•	eb.Entity.died (для игрока)
•	eb.Combat.decision.dialogic_made
Испускает:
•	EventBus.Game.state_changed(new_state, old_state)
•	EventBus.Game.paused(is_paused)

Entity.gd
Назначение: Базовый класс для всех живых сущностей (Player, Actor, враги). Хранит ссылки на компоненты (здоровье, ресурсы, прогресс, способности, Hurtbox), управляет их сигналами и ретранслирует события в EventBus.
Классы/Секции:
Ссылки на компоненты:
•	health_component: HealthComponent — здоровье
•	mana_component: ResourceComponent — мана
•	stamina_component: ResourceComponent — выносливость
•	progression_component: ProgressionComponent — уровень/опыт/статы
•	ability_component: AbilityComponent — способности
•	hurtbox: Hurtbox — область получения урона
Переменные состояния:
•	is_dead: bool — флаг смерти
•	movement_locked: bool — блокировка движения
•	interaction_locked: bool — блокировка взаимодействия
Методы:
•	func _ready() -> void — поиск компонентов, подключение сигналов, добавление в группу "entities"
•	func _find_components() -> void — ищет дочерние узлы-компоненты
•	func _setup_component_connections() -> void — подписка на сигналы компонентов (health, mana, stamina, progression)
•	func _on_health_changed(new_value, old_value, max_value) -> void — ретрансляция в EventBus.Entity.health_changed
•	func _on_died() -> void — установка is_dead, эмит EventBus.Entity.died, отключение сущности
•	func _on_damage_taken(amount, damage_type, source, is_critical) -> void — ретрансляция в EventBus.Entity.damage_taken
•	func _on_mana_changed(...) / _on_stamina_changed(...) -> void — ретрансляция изменений ресурсов
•	func _on_level_up(...) / _on_experience_gained(...) -> void — заглушки (реализация в Player)
•	func _disable_entity() -> void — выключает обработку, скрывает сущность
Публичные методы:
•	func take_damage(amount, damage_type, source = null, is_critical = false) -> void — наносит урон через HealthComponent
•	func heal(amount: int) -> void — лечение
•	func use_stamina(amount: int) -> bool
•	func use_mana(amount: int) -> bool
•	func get_stat(stat_name: String) -> int — запрос стата через ProgressionComponent
•	func is_alive() -> bool
•	func lock_movement(locked: bool = true) -> void
•	func lock_interaction(locked: bool = true) -> void
•	func _on_interaction_started(activator: Node) -> void — виртуальный, эмитит EventBus.Entity.interaction_started
•	func _on_interaction_ended() -> void — виртуальный, эмитит EventBus.Entity.interaction_ended
•	func get_horizontal_facing_direction() -> Vector2 — виртуальный, по умолчанию RIGHT
Сохранение:
•	func get_save_data() -> Dictionary — собирает уровень, опыт, HP/MP/стамину
•	func load_save_data(data: Dictionary) -> void — виртуальный, переопределяется в наследниках
Испускает EventBus:
•	EventBus.Entity.health_changed, died, damage_taken, mana_changed, stamina_changed, interaction_started, interaction_ended

EntityData.gd
Назначение: Ресурс данных сущности. Хранит уровень, опыт, HP/MP/стамину, базовые статы и активные эффекты. Предоставляет безопасные сеттеры и локальные сигналы для компонентов.
Экспортируемые переменные:
•	entity_name: String — имя сущности
•	level: int
•	experience: int
•	experience_to_next_level: int
•	max_hp / current_hp: int — здоровье
•	max_mp / current_mp: int — мана и её регенерация mp_regen_per_second
•	max_stamina / current_stamina: int — выносливость и её реген stamina_regen_per_second
•	base_stats: Dictionary — базовые характеристики (attack, defense, speed и т.п.)
•	active_effects: Array[Dictionary] — активные баффы/дебаффы
Локальные сигналы:
•	health_changed(new_value, old_value, max_value)
•	mana_changed(new_value, old_value, max_value)
•	stamina_changed(new_value, old_value, max_value)
•	stat_changed(stat_name, new_value)
•	level_changed(new_level, old_level)
•	experience_changed(new_exp, old_exp)
•	died
Аккумуляторы регенерации:
•	_mp_regen_accum: float
•	_stamina_regen_accum: float
Безопасные сеттеры:
•	func set_current_hp(value: int) -> void — clamp, сигнал health_changed, emit died при 0
•	func set_current_mp(value: int) -> void — только если max_mp > 0
•	func set_current_stamina(value: int) -> void — только если max_stamina > 0
•	func set_stat(stat_name: String, value: int) -> void — обновляет base_stats и сигнал stat_changed
•	func set_level(value: int) -> void
•	func set_experience(value: int) -> void
Управление ресурсами:
•	func use_stamina(amount: int) -> bool
•	func use_mana(amount: int) -> bool
•	func use_health(amount: int) -> bool — здоровье как ресурс (не даёт умереть от стоимости)
•	func regenerate_resources(delta: float) -> void — накопительная регенерация
•	func can_afford(stamina_cost: int = 0, mana_cost: int = 0, health_cost: int = 0) -> bool
Геттеры:
•	func get_stat(stat_name: String) -> int
•	func is_alive() -> bool
•	func get_hp_percentage() / get_mp_percentage() / get_stamina_percentage() -> float
Сохранение:
•	func get_save_data() -> Dictionary — полная снимок ресурсов и статов
•	func load_save_data(data: Dictionary) -> void — загрузка с клонированием словарей/массивов

Player.gd
Назначение: Конкретная реализация сущности игрока. Отвечает за движение в мире, интеграцию с боевой FSM, проигрывание анимаций, работу с PlayerData и взаимодействие с EventBus.
Ссылки:
•	animation_player: AnimationPlayer
•	sprite: Sprite2D
•	controller: PlayerController — обработчик ввода
•	combat_component: PlayerCombatComponent — боевой компонент
•	player_data: PlayerData — данные игрока
•	entity_data: EntityData (совместимость, маппится на player_data)
Переменные:
•	move_speed, sprint_speed, acceleration, friction — параметры движения
•	current_animation: String — текущая анимация
•	last_movement_direction: Vector2 — последнее направление
•	last_horizontal_direction: Vector2 — последний горизонтальный вектор
•	_can_move_in_world: bool — можно ли двигаться в WORLD
•	_in_combat_mode: bool — управляется ли движением FSM
Методы:
•	func _ready() -> void — вызывает Entity._ready(), настраивает группы, Hurtbox, подписывается на EventBus.Game.state_changed и EventBus.Animations.requested, подписывает прогрессию
•	func _physics_process(delta: float) -> void — движение в мире, если не в бою
•	func _get_input_vector() / _handle_movement() / _handle_stop_movement() — логика движения и анимаций
•	func get_horizontal_facing_direction() -> Vector2 — приоритет по velocity, иначе last_horizontal_direction
Анимации:
•	_play_walk_animation(direction: String)
•	_play_idle_animation()
•	_on_animation_requested(target, animation_name, duration) — обработка глобальных анимаций
Обработка состояний:
•	_on_game_state_changed(new_state: int, _old_state: int) -> void — включает/выключает движение, боевой режим, взаимодействия
Специфичные сигналы игрока:
•	_on_level_up(new_level, stat_increases) — эмит EventBus.Player.level_up
•	_on_experience_gained(amount, new_total, _next_level) — эмит EventBus.Player.experience_gained
Публичные методы:
•	force_stop() — принудительная остановка
•	teleport(new_position: Vector2, _fade_effect: bool = true)
•	set_last_horizontal_direction(dir: Vector2)
Сохранение:
•	get_save_data() — добавляет позицию, направление и текущую анимацию к данным Entity

PlayerController.gd
Назначение: Обработчик ввода игрока. Разделяет глобальный ввод (интеракции, меню) и боевой ввод (атаки, блок, уклонения, способности). Работает с EventBus и боевой системой.
Ссылки:
•	player: Player — владелец
Методы:
•	func _input(event: InputEvent) -> void — точка входа для ввода
•	func _handle_global_input(event: InputEvent) -> bool — обработка «interact» и «ui_cancel»
•	func _handle_combat_input(event: InputEvent) -> void — обработка боевых действий:
	- ЛКМ (basic_attack) → EventBus.Combat.attack.basic_requested или подтверждение каста
	- block press/release → EventBus.Combat.block.started/ended
	- dodge → EventBus.Combat.dodge.requested с направлением
	- ability_1..4 → EventBus.Combat.ability.slot_pressed
•	func _handle_interaction() -> void — эмит EventBus.Actors.interaction_requested
•	func _confirm_ability_cast() -> void — вычисляет глобальную позицию мыши по камере и эмитит EventBus.Combat.ability.target_confirmed
•	func _get_input_vector() -> Vector2 — вектор движения по WASD
•	func _can_open_menu() -> bool — меню доступно, если игрок может двигаться в мире или в бою

PlayerCombatComponent.gd
Назначение: Специализация CombatComponent для игрока. Подписывается на EventBus.Combat.* и прокидывает команды во внутреннюю FSM.
Особенности:
• Наследуется от CombatComponent и переопределяет _setup_connections().
Методы:
•	func _setup_connections() -> void — вызывает super, затем подключает:
	- EventBus.Combat.attack.basic_requested → _on_attack_requested
	- EventBus.Combat.dodge.requested → _on_dodge_requested
	- EventBus.Combat.block.started / .ended → _on_block_started / _on_block_ended
	- EventBus.Combat.ability.slot_pressed → _on_ability_slot_pressed (подтверждает подключение)

Actor.gd
Назначение: Конкретная реализация актора (NPC/врага). Наследуется от Entity, добавляет режимы (world/battle), работу с диалогами, взаимодействиями, анимациями и AI.
Экспорт:
•	actor_id: String — идентификатор актора
•	initial_mode: String — начальный режим (world/battle)

Ссылки:

Назначение: Состояние атаки в бою. Управляет шагами комбо, спавном Hitbox, окнами для продолжения комбо, переходами в другие состояния.

•	_handle_choice(enemy, choice) — маршрутизирует в _handle_attack_choice/_handle_dialogue_choice/_handle_spare_choice
•	_handle_attack_choice() — возвращает врага в enemies и режим battle
•	_handle_dialogue_choice() — эмитит Combat.decision.transition_to_dialogue для CombatManager
•	_handle_spare_choice() — переводит врага в режим world, помечает "spared" и эмитит Combat.decision.enemy_spared
•	_exit_decision_point() — сброс флагов, эмитит Game.decision_point_deactivated и Combat.decision.ui_closed

Назначение: Автозагрузка, управляющая боями: запуск/завершение, инициализация врагов, переходы в диалоги/мир, расчёт опыта.

•	_initialized_enemies: Array
